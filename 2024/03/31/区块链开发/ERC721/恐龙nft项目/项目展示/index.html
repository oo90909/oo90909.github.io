一：系统可运行展示

随意打开一个装有hardhat的项目，用npx hardhat node命令打开本地节点


打开metamask钱包，导入本地链给的账户私钥，可以看到有很多测试比


打开前端代码仓库，用命令 npm run start启动3000端口


等待一会以后网页弹出

部署合约，将合约地址复制到前端代码中相应位置

再展示前端和合约交互的代码

三：系统典型过程展示
恐龙交配和恐龙购买和龙蛋孵化：

  1. 首先要确保输入的两只恐龙种类相同
    来查看恐龙种类：


    可以看到5，6号的物种都是翼龙，且性别相反，所以可以交配

成功响应

错误处理：如果此时不放入规定的恐龙会发生什么呢？

1，2号是不同种类的恐龙


最后发现控制台没有任何反应


这是两只恐龙爱情的结晶，可以看到它的信息，是个翼龙

现在我们点击孵化，孵化的过程是100s是，这可能不符合常理

这个期间内，我们可以upload或者购买

趁这个时间我们去逛逛商城

我们进入了没有恐龙蛋的商城，在这里会报错：你没有恐龙蛋，这是合约里面的require返回的信息


我们来到恐龙商城，我刚刚偷偷upload了几只恐龙，


买回来，现在恐龙的价格都还是固定的，10000wei

我又生了一个龙蛋

现在取走龙蛋，看看有什么变化

少了一个龙蛋，这是因为龙蛋付出小恐龙了

多了一只小翼龙，是刚才那两只恐龙的宝宝
